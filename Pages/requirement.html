1. User Story Workshop

A user story is a simple, powerful way to describe a requirement from the user's perspective. It forces us to think about *who*, *what*, and *why*.

The Format

As a [type of user], I want to [perform some action], so that I can [achieve some goal].

Let's Practice!

Example:

As a Student,

I want to search for items using keywords,

so that I can quickly find my lost water bottle.

2. MoSCoW Prioritization Simulator

You can't build everything at once. The MoSCoW method helps teams decide what's critical for a release. Drag and drop the requirements into the correct buckets.

M - Must Have

Critical for launch. Non-negotiable.

S - Should Have

Important, but not critical. Can be delayed.

C - Could Have

Desirable but not necessary. "Nice to have".

W - Won't Have (This time)

Will not be included in this release.

3. Non-Functional Requirements (NFR) Explorer

NFRs define *how well* the system should work. We can use the FURPS+ model to explore them. Think about these questions for our Lost & Found portal.

Usability

How easy is it to use? A first-time user should be able to report a lost item in under 2 minutes.

Reliability

How often does it fail? The system should be available 99.9% of the time.

Performance

How fast is it? Search results should appear in under 2 seconds, even with 1000 items in the database.

Supportability

How easy is it to maintain? Code must be well-commented and follow a consistent style guide.

Security (+)

How secure is it? User passwords must be hashed and salted, not stored as plain text.

Interface (+)

How does it interact with other systems? The app must have an API to allow a future mobile app to connect.

4. Acceptance Criteria Builder

Acceptance Criteria (AC) make a User Story testable. They define the "Definition of Done". We use the Gherkin syntax (Given-When-Then) for this.

The Format

Given [a context or precondition]

When [I perform an action]

Then [I should see an expected outcome]

Let's Build AC for a User Story

User Story:

As a Student, I want to search for items using keywords, so that I can quickly find my lost water bottle.

Example AC:

Scenario: Successful Search

Given I am on the homepage,

When I type "bottle" into the search bar and press Enter,

Then I should be taken to the search results page and see a list of items containing the word "bottle".
